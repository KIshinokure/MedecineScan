# MedecineScan (Variant 25)

Веб-сервис для классификации медицинских изображений (рентген, КТ) с применением сверточных нейросетей (PyTorch). Сервис возвращает класс патологии и тепловую карту (Grad-CAM), визуализирующую области внимания модели для объяснимости принятых решений.

## Описание проекта

Данный проект представляет собой специализированный медицинский Web-API сервис, предназначенный для автоматического анализа радиологических снимков. С помощью глубоких сверточных нейросетей система определяет тип патологии и генерирует визуальное обоснование диагноза.

### Ключевые особенности
* ✅ **Объяснимый ИИ (XAI)** — наложение тепловой карты Grad-CAM для демонстрации зон внимания нейросети.
* ✅ **Высокая производительность** — асинхронная обработка входящих запросов средствами FastAPI.
* ✅ **Строгая валидация** — проверка типов и размеров файлов "на лету" с помощью Pydantic V2.
* ✅ **Контейнеризация** — готовая конфигурация Docker и Docker Compose для быстрого развертывания.
* ✅ **Контроль качества** — полное покрытие модульными тестами и статический анализ уязвимостей.

Функциональные возможности
1. Сервисные эндпоинты
GET /health — проверка жизненного цикла API и статуса инициализации весов нейросети.

GET /docs — интерактивная графическая документация Swagger UI.

GET /openapi.json — сырая спецификация схемы OpenAPI.

2. Подсистема классификации снимков
POST /predict — основной аналитический метод. Принимает бинарный файл снимка, проверяет его MIME-type и передает в граф вычислений PyTorch. Возвращает предсказанный класс патологии, метрику уверенности и готовое изображение с картой внимания Grad-CAM.

Быстрый старт
Развёртывание через Docker
Bash

docker compose up --build

# Запуск в фоновом режиме
docker compose up -d

После старта веб-сервис доступен по адресу: http://localhost:8000

API Reference
POST /predict
Отправка медицинского изображения на анализ.

Method: POST

Content-Type: multipart/form-data

Payload: file (Объект изображения: jpeg, png)

Response (200 OK):

JSON
{
  "pathology_class": "Pneumonia",
  "confidence": 0.94,
  "grad_cam_base64": "iVBORw0KGgoAAAANSUhEUgAA..."
}
Error Responses:

400 Bad Request — отправленный файл не является поддерживаемым типом изображения.

422 Unprocessable Entity — не передан обязательный параметр file или тело запроса повреждено.

500 Internal Server Error — сбой на этапе матричных вычислений модели или генерации тепловой карты.

Сценарии использования
Полный поток выполнения запроса
Пользователь (врач) загружает снимок легких через интерфейс или стороннюю интеграцию.

Система принимает multipart-запрос на эндпоинт POST /predict.

Pydantic валидирует структуру метаданных, а OpenCV декодирует байты в BGR-матрицу.

Нормализованный тензор передается в сверточную сеть PyTorch для расчета логитов классов.

На основе градиентов последнего сверточного слоя модуль grad-cam строит маску активации.

Маска накладывается на исходный снимок, конвертируется в Base64-строку и возвращается клиенту.

Структура данных
Модель ответа (PredictionResponse)
JSON
{
  "pathology_class": "строка (название найденной патологии)",
  "confidence": "число с плавающей точкой (степень уверенности модели от 0.0 до 1.0)",
  "grad_cam_base64": "строка (результирующее изображение в кодировке Base64)"
}
