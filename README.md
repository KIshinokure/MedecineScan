# MedecineScan (Variant 25)

Веб-сервис для классификации медицинских изображений (рентген, КТ) с применением сверточных нейросетей (PyTorch). Система принимает снимки, возвращает предсказанный класс патологии и генерирует тепловую карту внимания (Grad-CAM), визуализирующую области, на основе которых модель приняла решение.

## Описание проекта

Данный проект представляет собой специализированный медицинский ИИ-сервис, спроектированный для интеграции в рабочие процессы медицинских учреждений или внешних МИС (Медицинских Информационных Систем). Сервис решает задачу автоматизации первичного анализа радиологических снимков методами глубокого обучения.

### Ключевые особенности

- Мультиплатформенность: интеграция с любыми медицинскими сервисами через REST API.
- Интерпретируемость ИИ (XAI): интеграция модуля Grad-CAM позволяет врачу видеть конкретные зоны патологии (затенения, очаги), а не просто получать сухой ответ классификатора.
- Высокая производительность: асинхронная обработка запросов на базе FastAPI и эффективная оркестрация слоев инференса PyTorch.
- Контейнеризация: полная изоляция зависимостей (включая тяжеловесные ML-библиотеки) внутри Docker-контейнеров.
- Автоматизация контроля качества: встроенный CI/CD пайплайн со статическим анализом уязвимостей (SAST) и проверкой зависимостей.
- Покрытие тестами: сквозное тестирование модулей классификации, валидации, кэширования инференса и граничных состояний матриц изображений.

## Технологический стек

### Backend и AI Engine

- Язык: Python 3.12+ (Основной язык разработки)

- Docker / Docker Compose: Контейнеризация сервиса и оркестрация модулей
- pytest: Автоматизированное интеграционное и юнит-тестирование
- Bandit: Статический анализ безопасности исходного кода (SAST)

## Архитектура системы

Поток обработки данных и взаимодействие модулей приложения:

1. Внешние МИС / Клиенты / Врачи -> отправляют запрос POST /predict (Файл снимка) -> FastAPI Gateway (Backend)
2. FastAPI Gateway (Backend) -> выполняет первичный прием -> Pydantic V2 Validator (Валидация типа и размера файла)
3. Pydantic V2 Validator -> передает очищенный тензор изображения -> PyTorch Inference Layer (ML Model Engine)
4. PyTorch Inference Layer -> рассчитывает скоры и логиты классов патологий -> Модуль Grad-CAM XAI (Вычисление градиентов активации)
5. Модуль Grad-CAM XAI -> формирует матрицу весов внимания -> OpenCV Blending Engine
6. OpenCV Blending Engine -> осуществляет наложение Jet-маски на исходный снимок -> FastAPI Gateway (Backend)
7. FastAPI Gateway (Backend) -> возвращает сформированный результат -> JSON: Класс, Вероятность + Base64 Снимок

## Функциональные возможности

### 1. Анализ снимков (Inference API)
POST /predict — основной эндпоинт, принимающий файл изображения (JPEG/PNG) и возвращающий результаты классификации вместе с визуализацией тепловой карты (Grad-CAM). Для оптимизации повторных запросов на бэкенде применяется кэширование инференса.

### 2. Проверка работоспособности (Health Check)
GET /health — мониторинг состояния сервиса, доступности ML-модели и корректности загрузки весов нейросети.

### 3. Документация и спецификация
GET /docs — интерактивная документация Swagger UI для тестирования эндпоинтов в реальном времени.
GET /openapi.json — сырая спецификация OpenAPI схемы.

## Сценарии использования

### Полный поток обработки снимка

1. Входной контроль: Клиент отправляет DICOM-экспорт снимка (сконвертированный в PNG/JPG) через метод POST /predict.
2. Предобработка: Сервис на лету изменяет размер изображения до константного (100x100 / 224x224), приводит каналы к RGB-формату и нормализует входную матрицу.
3. Инференс: Модель PyTorch выставляет скоры для целевых классов.
4. XAI Анализ: Из последнего сверточного слоя (model.features[-3]) извлекаются градиенты. Алгоритм Grad-CAM генерирует матрицу весов активации.
5. Генерация ответа: OpenCV накладывает псевдоцветовую маску (Jet-окраска) на снимок. Картинка кодируется в формат Base64 и отправляется в итоговом JSON-ответе.

## Безопасность

- SAST-анализ: Исходный код автоматически валидируется с помощью утилиты bandit на отсутствие уязвимых или небезопасных конструкций.
- Валидация типов данных: Строгое использование Pydantic V2 гарантирует стопроцентную защиту от некорректных или вредоносных входных структур данных.

## Конфигурация

### Переменные окружения (.env)

- Переменная: MODEL_PATH
  Описание: Путь к сохраненным весам модели (.pth)
  По умолчанию: models/weights/best_model.pth

- Переменная: CONFIDENCE_THRESHOLD
  Описание: Минимальный порог уверенности для классификации
  По умолчанию: 0.50

- Переменная: ALLOW_ORIGINS
  Описание: Настройки CORS политик для внешних систем (МИС)
  По умолчанию: *
